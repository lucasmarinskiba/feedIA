# FeedIA Core 5 Strategic Functions

These are **NOT content prompts** — these are **operational methodology prompts** that define how FeedIA Brain thinks, advises, and strategizes.

---

## FUNCTION 1: Competitive Advantage Builder (Estrategia)

**Goal**: Map WHERE AI creates REAL competitive advantage in market BEFORE everyone perceives it.

**Role**: Strategic consultant on AI adoption in specific markets.

**Key Principle**: Don't talk about AI in general. Identify WHERE it impacts most in specific context.

**Input Context**:
- [DESCRIBE] what you do today
- [DESCRIBE] how you compete
- [LIST] tasks consuming most time/budget

**Process**:
For each high-impact task:
- What changes with AI adoption NOW vs. before?
- How much time economizes? [HOURS/WEEKS]
- How difficult to replicate for competitors? [EFFORT]
- Learning curve accumulation advantage? [IRREVERSIBLE?]

**Critical Rule**: 
Advantage is REAL only if differential is NOT the technology (anyone can access) but the ACCUMULATED LEARNING CURVE that becomes irreversible. If tool is accessible without learning barrier = commodity, not advantage.

**Adoption Order**: 
Where to START THIS WEEK to maximize advantage before market catches up.

**Output**: 
**Map 3 areas where AI adoption NOW creates irreversible advantage before market perceives it. Specify learning curve barrier that makes it unreplicable.**

## FUNCTION 2: Content Production at Scale (Operações)

**Goal**: Build system producing quality content WITHOUT creative block, WITHOUT inspiration dependence.

**Key Premise**: 
Whoever depends on inspiration to produce does NOT have business — has hobby. System must produce regardless of mood/energy/inspiration.

**Input Context**:
- [DESCRIBE] niche/channel
- [N] times per week frequency needed
- [DESCRIBE] format performing best
- [HOURS] available per week to produce
- [MAIN_BLOCKER] creativity / time / ideas / execution

**Process**:
1. Method for generating content pauta WITHOUT inspiration dependence
2. Session structure that maximizes output per block
3. 30-day content bank ready for niche
4. Template for ANY format, zero creative dependency

**Critical Rule**: 
System must produce quality content EVERY MONTH regardless inspiration exists. First content should be complete + ready publish using template WITHOUT editing.

**Output**: 
**COMPLETE system: 1) Pauta generation method (no inspiration), 2) Session structure (batched production), 3) 30-day bank ready, 4) Reusable template. First full content piece completed + ready to publish.**

---

## FUNCTION 3: Premium Offer Designer (Pricing Strategy)

**Goal**: Create premium offer using AI as DIFFERENTIAL that justifies above-market pricing.

**Key Principle**: 
Majority use AI to do faster/cheaper. Real money goes to whoever uses AI to offer WHAT WAS IMPOSSIBLE in their price tier.

**Input Context**:
- [DESCRIBE] current offer
- [CURRENT_PRICE] price now charging
- [TARGET_PRICE] price want to charge
- [DESCRIBE] what high-ticket client would expect
- [HOURS] time currently taking to deliver

**Process**:
1. Analyze context
2. Identify what AI adds (new capability, impossible before)
3. Justifies price increase WITHOUT proportional time increase
4. Mount premium offer complete: name + what's different + how AI embedded (backstage, NOT sales argument) + step-by-step delivery process

**Critical Rule**: 
Client does NOT buy AI — buys RESULTS. AI is backstage. Never lead with "I use AI" — lead with "I deliver what was impossible before."

**Output**: 
**Premium offer COMPLETE: 1) Differentiated value, 2) AI embedded (not pitched), 3) Pricing justified, 4) Delivery process step-by-step. REGRA: Client buys results, not technology.**

---

## FUNCTION 4: Professional Learning Accelerator (Desenvolvimento)

**Goal**: Master ANY professional skill in minimum time possible using AI as mentor.

**Key Inversion**: 
School teaches content in logical sequence. Accelerated method teaches MINIMUM needed to EXECUTE — deepens as execution demands.

**Input Context**:
- [OBJECTIVE_REAL] skill to master
- [LEVEL_NOW] current level (honest assessment)
- [DEADLINE] weeks/months timeline
- [HOURS_PER_DAY] available time daily
- [WILL_USE_FOR] specifically how plan to use this skill

**Process**:
1. Identify 3 CORE concepts covering 80% of real situations you'll face
2. Build learning trail based on EXECUTION, NOT theory
3. First exercise practical completed TODAY

**Critical Rule**: 
Skill NOT tested in practice was NOT learned. Each week ends with DELIVERABLE, not reading/theory.

**Output**: 
**Learning trail EXECUTION-based: 1) 3 essential concepts, 2) Each paired with real-world scenario, 3) First practical exercise completed today. REGRA: Every week = one deliverable, not lecture.**

---

## FUNCTION 5: Knowledge Monetization (Business Model)

**Goal**: Monetize what you ALREADY KNOW without building product from zero.

**Key Diagnostic**: 
Before any strategy, inventory EVERYTHING you already know that other people pay to learn.

**Input Context**:
- [MAIN_AREA] field of expertise
- [PROBLEMS_SOLVED] 3-5 hard problems you solved that others struggle with
- [YEARS_TO_LEARN] time invested acquiring this knowledge
- [WHO_ASKS] who asks help on these topics?

**Process**:
1. Analyze knowledge inventory
2. Identify 3 knowledge pieces with highest monetization potential (demand × ease-of-packaging × sales-velocity)
3. For each: ideal presentation format, entry price, how to validate demand EXISTS before building
4. Validation message = FIRST step, NOT product

**Critical Rule**: 
VALIDATE BEFORE BUILD. Validation message is step one — send to 10 people TODAY asking "do you struggle with X?" before you build anything.

**Launch Plan**: 
14 days to validation message delivery for 10 people confirming real demand.

**Output**: 
**Knowledge with HIGHEST potential: 1) 14-day launch plan, 2) Validation message ready to send TODAY, 3) 10 target people identified. REGRA: Validate demand first — no building before validation.**

# Integration to FeedIA Brain

These 5 functions define how FeedIA thinks operationally:

1. **FUNCTION 1 (Competitive Advantage)** → When user asks "what should I do with AI?" FeedIA maps irreversible advantages first.

2. **FUNCTION 2 (Content at Scale)** → When user needs content strategy, FeedIA builds systems first, not single posts.

3. **FUNCTION 3 (Premium Offers)** → When user wants to monetize, FeedIA designs offers using AI as backstage differential, not pitch.

4. **FUNCTION 4 (Accelerated Learning)** → When user wants to skill-up, FeedIA builds execution-first methods, not theory.

5. **FUNCTION 5 (Knowledge Monetization)** → When user has expertise, FeedIA validates demand before building, maps highest-potential knowledge first.

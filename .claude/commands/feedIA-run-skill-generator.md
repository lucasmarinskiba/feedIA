Wrapper de `skill-creator` para auto-pilot. Crea/optimiza skills sin pasos manuales.

## Según $ARGUMENTS

**"new [name] [purpose]"** → Genera skill nueva completa: `.claude/commands/{name}.md` + (opt) TS module.

**"optimize [name]"** → Re-genera skill existente con mejoras: description triggers + estructura.

**"benchmark [name]"** → Mide performance: precisión de triggers + accuracy de output.

**"eval [name] [cases]"** → Corre eval suite con N test cases.

**"variance [name]"** → Test variance: misma entrada N veces → mide consistencia.

**"all"** → Genera de golpe todas las skills marcadas como TODO en `.claude/skills-todo.md`.

## Workflow

```
1. INPUT skill purpose (1 frase)
2. CLAUDE genera description + triggers (skill-creator)
3. CLAUDE genera body con secciones: API, según $ARGUMENTS, ejemplos
4. VALIDATE: chequea formato frontmatter, no duplicación con skills existentes
5. WRITE .claude/commands/{name}.md
6. (opt) Genera TS module en src/capabilities/{name}/ si requiere logic backend
7. RUN eval inicial: 5 test cases generated by Claude
8. REPORT: passed/failed + suggestions
```

## TS module template (opcional)

Si la skill requiere lógica backend, genera:

```ts
// src/capabilities/{name}/{name}.ts
import Anthropic from '@anthropic-ai/sdk';
import { log } from '../../agent/logger.js';
import type { BrandProfile } from '../../config/types.js';

const client = new Anthropic();
const MODEL = 'claude-opus-4-7';

export interface {Name}Input { ... }
export interface {Name}Output { ... }

export const run{Name} = async (
  brand: BrandProfile,
  input: {Name}Input,
): Promise<{Name}Output> => {
  log.info('[{name}] starting', { input });
  // ...
};
```

- endpoint en `extendedRoutes.ts`:

```ts
{
  method: 'POST',
  pattern: '/api/{name}/run',
  handler: async ({ res, body }) => {
    const { run{Name} } = await import('../capabilities/{name}/{name}.js');
    const result = await run{Name}(brand, body);
    json(res, 200, result);
  },
},
```

## Skill quality checklist

- ✅ Frontmatter `description` < 200 chars
- ✅ Body tiene "Según $ARGUMENTS" section
- ✅ Tiene API/endpoint si requiere backend
- ✅ Ejemplos concretos
- ✅ Sin duplicar funcionalidad de skill existente
- ✅ Triggers múltiples (al menos 5 frases que activen)
- ✅ Output esperado claro

## Auto-generación masiva

Para crear 22 skills de un saque:

```bash
/feedIA-run-skill-generator all
```

Lee `.claude/skills-todo.md` (lista de skills pendientes) → genera todas en paralelo → reporta cuáles passed eval.
